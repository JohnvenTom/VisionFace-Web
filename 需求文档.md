基于YOLOv11的Web人脸检测系统项目开发需求文档

## 1. 项目概述

### 1.1 项目背景

人脸检测是计算机视觉最基础、最核心的应用场景，广泛应用于智能监控、门禁识别、人机交互等领域。传统桌面端人脸检测程序交互性差、展示效果弱、部署不便。本项目基于**YOLOv11开源预训练人脸检测模型**，结合轻量后端与前端页面，搭建一套浏览器端可直接使用的人脸检测 Web 系统，支持图片上传检测、本地摄像头实时人脸检测，具备可视化检测结果、数据统计、参数可调等功能，适合课程设计、毕业设计与技术演示落地。

### 1.2 项目目标

搭建一套**轻量、易用、可演示、可拓展**的 Web 人脸检测系统，无需客户端安装，浏览器直接访问即可使用，依托 YOLOv11 高精度、低参数量、高推理速度的优势，实现对正面、侧脸、轻度遮挡、远距离小人脸的稳定检测，同时提供完整的工程化架构，支持后续拓展人脸识别、人脸关键点检测、视频流批量检测功能。

### 1.3 适用范围

本文档定义系统全部功能需求、性能需求、架构规范、开发标准、验收标准，适用于本项目开发、测试、部署、答辩及文档撰写。

## 2. 整体系统架构

系统采用**前后端分离轻量架构**，三层结构：前端展示层 + 后端服务层 + YOLOv11推理核心层，模块解耦、可独立替换、易于维护。

### 2.1 架构分层

1. **前端展示层**：HTML+CSS+JavaScript，负责页面展示、图片上传、摄像头调用、参数配置、检测结果可视化展示。
2. **后端服务层**：FastAPI 轻量接口服务，接收前端图像数据、调用推理模块、返回检测结果、提供跨域支持。
3. **AI推理核心层**：基于 Ultralytics YOLOv11 预训练人脸模型，完成图像预处理、推理、NMS、后处理结果解析。

### 2.2 核心流程

用户操作（上传图片/打开摄像头）→ 前端图像编码传输 → 后端接收数据 → 图像预处理 → YOLOv11 推理检测 → 结果解析（人脸框、置信度、人脸数量）→ 后端返回JSON结果 → 前端绘制标注、展示统计信息。

## 3. 技术栈定义

### 3.1 后端技术栈

- 编程语言：Python 3.9+
- Web框架：FastAPI（高性能、轻量、接口简洁）
- AI框架：Ultralytics（YOLOv11 官方库）
- 图像处理：OpenCV-Python、PIL
- 数据处理：Numpy、Base64 编解码
- 跨域支持：CORS 中间件

### 3.2 前端技术栈

- 基础：HTML5 + CSS3 + JavaScript
- UI 样式：原生CSS/简易Bootstrap，简洁现代化界面
- 设备能力：Navigator.mediaDevices 调用本地摄像头
- 可视化：Canvas 绘制人脸检测框与置信度文本

### 3.3 核心模型

- 模型：YOLOv11n-face 预训练权重（轻量极速版，适配PC端CPU实时推理）
- 优势：相比YOLOv8，参数更少、小目标人脸更准、遮挡鲁棒性更强、CPU推理速度更快

## 4. 功能需求（核心必做功能）

### 4.1 前端功能需求

#### 4.1.1 图片上传检测功能

- 支持本地 jpg、png、jpeg 图片上传
- 上传后自动向后端发送检测请求
- 前端接收结果，在图片上绘制人脸矩形框、显示置信度分数
- 展示本次检测**人脸总数**统计信息

#### 4.1.2 实时摄像头检测功能

- 浏览器调用本地摄像头，实时视频流展示
- 逐帧传输后端推理，实时绘制人脸检测结果
- 支持开启/关闭摄像头、暂停检测
- 实时展示当前画面人脸数量

#### 4.1.3 参数可调功能

- 置信度阈值调节（默认0.5，范围0.1–0.9）
- NMS非极大值抑制阈值调节（默认0.45）
- 参数修改实时生效，无需重启服务

#### 4.1.4 结果展示与重置

- 清晰展示检测后效果图、检测时间、人脸数量
- 支持一键清空画面、重置检测状态

### 4.2 后端功能需求

#### 4.2.1 模型加载模块

- 服务启动一次性加载 YOLOv11n-face 预训练模型，常驻内存，避免重复加载耗时
- 自动适配CPU/GPU推理设备

#### 4.2.2 图片检测接口

- 接收前端 Base64 图片数据、置信度、NMS参数
- 完成图像预处理、YOLOv11推理、结果过滤
- 返回标准化JSON结果：人脸坐标、置信度、人脸总数、推理耗时

#### 4.2.3 视频帧推理接口

- 适配摄像头实时帧流，轻量快速推理
- 控制推理帧率，避免前端卡顿

#### 4.2.4 跨域与异常处理

- 全局CORS跨域配置，支持本地前端调试
- 对空数据、损坏图片、异常参数做容错处理，返回友好错误信息

## 5. 非功能需求

### 5.1 性能需求

- 普通笔记本CPU：单张图片推理耗时 < 100ms
- 摄像头实时检测帧率稳定 15–25FPS
- 支持同时检测多人、侧脸、轻度遮挡人脸

### 5.2 可用性需求

- 免安装、浏览器直接访问，开箱即用
- 启动命令简单，部署流程极简
- 代码模块化、注释完整、便于二次修改与拓展

### 5.3 兼容性需求

- 支持 Chrome、Edge 主流浏览器
- 支持 Windows、Mac、Linux 系统运行后端服务

## 7. 核心接口设计

### 7.1 图片检测接口

- 请求地址：/api/detect/image
- 请求方式：POST
- 请求参数：base64图像、conf阈值、iou阈值
- 返回数据：人脸框坐标列表、各人脸置信度、人脸总数、推理耗时

### 7.2 视频帧检测接口

- 请求地址：/api/detect/frame
- 请求方式：POST
- 功能：实时帧推理，适配摄像头流

## 8. 系统运行流程

1. 后端执行 main.py 启动 FastAPI 服务，加载 YOLOv11 人脸模型
2. 浏览器访问本地网页前端界面
3. 用户选择图片上传 / 开启摄像头实时检测
4. 前端传输图像数据至后端接口
5. 后端完成预处理、YOLOv11推理、结果筛选
6. 返回结构化检测数据，前端Canvas绘制框与文字
7. 页面展示统计结果与检测耗时

## 9. 验收标准

项目完成后满足以下全部条件即为验收通过：

1. 服务可正常启动，无报错，模型加载成功
2. 网页端可正常打开，界面整洁可用
3. 图片上传可精准检测人脸，支持多人脸检测
4. 摄像头实时检测流畅，无明显卡顿、无大量误检漏检
5. 支持置信度、NMS参数手动调节并生效
6. 可正常统计人脸数量、展示推理耗时
7. 代码结构规范、可运行、可复现
